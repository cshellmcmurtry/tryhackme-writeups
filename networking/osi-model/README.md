## Objective

This room introduced the Open Systems Interconnection (OSI) model and explained how it standardizes network communication by organizing networking functions into seven distinct layers. Understanding the OSI model provides a framework for troubleshooting, analyzing, and securing network communications.

## Key Concepts

### What is the OSI Model?

The OSI model is a conceptual framework that divides network communication into seven layers. Each layer has a specific responsibility and works together with the layers above and below it to transmit data between devices.

### The Seven Layers

1. **Physical Layer (Layer 1)** – Transmits raw bits across physical media such as cables or wireless signals.
2. **Data Link Layer (Layer 2)** – Handles communication between devices on the same local network using MAC addresses.
3. **Network Layer (Layer 3)** – Routes packets between networks using IP addresses.
4. **Transport Layer (Layer 4)** – Ensures reliable data delivery through protocols such as TCP and UDP.
5. **Session Layer (Layer 5)** – Establishes, manages, and terminates communication sessions.
6. **Presentation Layer (Layer 6)** – Translates, encrypts, and compresses data before it reaches the application.
7. **Application Layer (Layer 7)** – Provides network services directly to user applications.

### Why the OSI Model Matters

The OSI model provides a structured way to understand how data moves across a network. It also helps security professionals identify where communication issues or security events may occur.

### Troubleshooting with the OSI Model

Breaking network communication into layers makes troubleshooting more efficient. If an issue occurs, analysts can isolate the problem by determining which layer is affected.
